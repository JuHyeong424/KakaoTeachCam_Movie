깃허브 페이지
https://juhyeong424.github.io/KakaoTeachCam_Movie/

API_KEY는 현재까지 배운 내용에서 .env를 사용하지 못하여 public으로 두었습니다. 
혹시 다른 방법이 있다면 알려주시면 바로 적용하겠습니다.

1. Home 클릭시
   - 기본으로 제공하는 페이지의 1페이지로 이동한다.
2. 원하는 단어를 검색
   - 해당 단어를 포함한 결과가 나온다.
3. 페이지 이동
   - 기본에서는 최대 500페이지까지 검색할 수 있다.
   - 검색된 페이지에서는 데이터에서 나온 max_page까지 나올 수 있다.
   - 원하는 페이지를 검색할 수 있다.
4. 모달창
   - 카드를 누르면 영화 상세 페이지가 나온다.
   - 맨 오른쪽 위에 x를 누르거나 화면 밖을 누르면 모달창에서 나올 수 있다.
  
부족한 내용
1. **로컬 저장소 활용한 ‘북마크’ 기능**
- 관심 있는 영화들을 ‘북마크’ (혹은, ‘좋아요’) 할 수 있는 기능을 구현해보세요. 특정 버튼을 눌렀을 때, `localStorage` 에 관련 데이터를 저장하게 해봅시다.
  
2. **쓰로틀링/디바운싱** 
- 사용자가 검색어를 입력할 때마다 API를 호출하는 대신, **스로틀링** 또는 **디바운싱**을 사용해 **최적화된 API 호출**을 구현하세요.
- 이 과제는 `모든 필수 기능을 마친 후` 성능 최적화를 도전하고 싶다면 시도해 보세요.
- **중요 포인트**: 성능 최적화를 통해 불필요한 API 호출을 줄이는 것을 목표로 합니다.
